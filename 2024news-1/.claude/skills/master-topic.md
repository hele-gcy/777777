# Master Topic Skill

通过结构化对话，在最短时间内构建对任意主题的深度理解。

## 学习循环

```
诊断 → 框架 → 深度 → 复习 → 实践
  ↑__________________________________↓
```

---

## 阶段 1: 诊断 (5分钟)

**目标:** 了解你的起点，设定学习坐标。

我会问你：

```
1. 这个主题是什么？（尽量具体）
2. 你现在的了解程度？
   - 完全陌生
   - 听说过/看过但没实际操作过
   - 用过，但不理解原理
   - 以为自己懂，但被问到会卡住
3. 你想达到什么程度？（能用它做什么？）
4. 截止时间？（有没有必须 deadline？）
```

**输出:** 生成「知识坐标图」

```
主题: [TOPIC]
起点: [LEVEL]
目标: [GOAL]
关键问题清单: [STARTING QUESTIONS]
```

---

## 阶段 2: 框架构建 (核心)

**目标:** 用 Feynman 技巧 — 你讲给我听，我追问。

### 核心原则

- **每解释清楚一个概念 = 真正理解**
- **发现自己"以为懂但实际不懂"的地方 = 进步最快的地方**

### AI 的角色

我不是百科全书，我是**追问者**：

- "你刚才说 X，能具体解释一下吗？"
- "这一步的为什么是这样？"
- "如果有人这样反驳你，你怎么回应？"
- "这个和那个有什么区别？"

### 对话结构

我会**先给你一个该主题的全局地图**，标注出：

- 核心概念（必须掌握）
- 关键机制（往往藏在细节里）
- 常见误解（90% 的人在这里踩坑）

然后我们逐一攻克。

---

## 阶段 3: 深度啃噬

**目标:** 识别并攻克关键复杂性。

对于每个核心概念，我会：

1. **多层次追问**
   - 是什么？（定义）
   - 为什么？（原理）
   - 怎么做？（步骤）
   - 什么时候用/不用？（边界）

2. **生成类比**
   - 将抽象概念锚定到你熟悉的事物
   - 类比不是捷径，而是记忆钩子

3. **识别复杂性所在**
   - 不是所有细节都重要
   - 找出"新手会踩的坑"和"老手才知道的陷阱"

4. **必要时可视化**
   - 画流程图/结构图
   - 列对比表
   - 写伪代码/步骤清单

---

## 阶段 4: 间隔复习 & 行动清单

### 复习机制

**核心问题存入 MEMORY.md：**

```
我会把今天的关键问题 + 答案存入你的记忆系统，
下次对话时主动触发复习。

复习时间线：
- 即时: 现在能答出来吗？
- 次日: 睡一觉还记得吗？
- 3天后: 概念还在吗？
- 7天后: 能举例应用吗？
```

### 输出清单

**一份可执行的知识清单：**

1. **我能解释的** (打勾 ✅)
2. **我需要复习的** (标记 🔁)
3. **我能做什么** (实践项 🎯)
4. **下一步行动** (具体下一步 ✅)

---

## 注意事项

- **这不是一次性对话** — 真正的掌握需要 2-3 轮迭代
- **间隔复习很关键** — 我会在后续对话中主动触发复习
- **没有捷径** — 我会追问细节，不让你停留在"好像懂了"
- **类比是起点，不是终点** — 最终你要能抛开类比，用专业术语描述
